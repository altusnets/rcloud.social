The Big Picture

  • Why RCloud?

    1. Gain Greater Insights and Reduce Processing Time on Big Data Analyses through Collaboration

      2. RCloud enables Data Scientists to find value in data by sharing ideas and techniques with each other and interactive data analysis results with any user in the community through a conveniently accessible cloud computing platform.

      RCloud employs notebooks composed of cells where topics may easily be broken up into concepts and where the relationship between the concepts can be easily understood.

      In this environment, data scientist teams benefit from easier sharing of scripts and data feeds, experiments, annotations, and automated recommendations which are well beyond what traditional individual or locally based development environments provide. Rather than spending time "recreating the wheel", RCloud users may reuse parts of existing analyses which are recorded using built-in version control. Each analysis benefits from an ongoing iterative process to ensure accurate and appropriate techniques are being applied to move analyses forward.

      Additional functionality in RCloud notebooks versus other notebook environments includes authenticated security, built-in version control and the ability to search, comment and "like" (star) notebooks.

    2. Build Confidence with Reproducible Results

      3. Notebooks are project packages which contain all the components and dependencies of an analysis, including code, data, comments and other technical documentation, charts and deployment capabilities. User confidence is increased since results may be be verified and reproduced by anyone with access to the notebook.

    3. Increase Development Capabilities and User Engagement through the Cloud Platform

      4. As opposed to analysis which are stored locally, web-based content allows Data Scientists to collaborate over the Internet without concern for code or data becoming out of sync. An RCloud session runs on both client and server, so it is possible for R functions on the server to call JavaScript functions on the client, and vice versa. This means rich web-based content through easy integration of custom Javascript user interface (UI) and Javascript libraries such as JQuery and D3. In addition, since RCloud is platform independent and user access and controls remain constant, local development environment parameters no longer create hurdles for viewing and running analyses promoting user confidence and engagement.

    4. Scalable Architecture

      5. RCloud is an ideal environment for research and data analysis departments who commonly use R and/or Python to share, publish and archive their code and data. RCloud is roughly the same in terms of compute efficiency as in memory Business Intelligence (BI) tools like Tableau and Qlikview, but unlike these systems, RCloud is open-source software that may be installed locally to create private clouds where additional users may be added without licensing concerns. RCloud supports efficient, secure, client/server connections via the FastRWeb package and the "Object Capabilities" model, maintains an automatic git based trail of code modifications which document the development history of an RCloud based project, and it can also handle processing large data files in R via the iotools package. RCloud is an ideal platform to quickly develop interactive web based R and/or Python programs (especially if you are already an R/Python programmer) which may easily be shared with others.

    5. Transform Knowledge into Assets

      6. Code, widget, dashboard and analysis libraries are ancillary benefit that comes with the creation of every RCloud notebook (on the public instance or in private clouds). These knowledge assets may used on similar projects or to train new employees, for example. RCloud notebooks are searchable and forkable (sharable), so anyone with access to the notebooks may view and reuse various parts of a notebook and its content.

    6. Built-in Security

      7. RCloud has adopted the discipline known as Object-Capabilities (ocap) which creates a built-in security for client-server communication. This means that web browsers never directly instruct the RCloud backend to execute arbitrary code which prevents unauthenticated clients from making unauthorized calls to the RCloud runtime environment; read more about ocap in our Documentation or in Wikipedia. In addition, RCloud notebooks may be encrypted for added security. In the notebook product space, authenticated client-server channeling and notebook encryption are unique to RCloud.

    7. Minimize Development to Deployment Effort

      8. Standard analytic workflows often involve performing a data science experiment using one set of tools and deploying the results using other tools. Rather than stitching pipelines together and/or dealing with inconsistences between development and production environments, every RCloud notebook is named by a URL so analyses may easily be transformed from code and technical documenation to rich web-based dashboards.

  • Public versus Private RClouds (or RCloud.Social versus Download / Local Installation)

    • Public RCloud Instance

    The public RCloud instance is the perfect place to test drive RCloud. Anyone with a GitHub account can have full user access to this instance. If you don't already have a GitHub account you can create one at the GitHub login page; see the next section Data Scientist Access (or Logging in) for details.

    Private RClouds

    The RCloud server program can easily be installed locally on any Linux machine or on OSX via MacPorts. A local instance has the advantage of granting you complete control of your notebooks and data. Local instances also afford the opportunity to evaluate and experiment with the RCloud infrastructure.

    By default your local RCloud instance will still use GitHub for authentication. Before proceeding with an installation you will need to setup up a new 'OAuth' application in your GitHub profile. Here's how:

    1. Go to your GitHub profile page and select 'Edit Profile'.
    2. In the personal settings menu select 'OAuth applicatoins'.
    3. Select the 'Developer applications' tab.
    4. Press the 'Register a new application' button.
    5. Assuming that you are testing on your local IP address 127.0.0.1 and the default RCloud port (8080) is available fill in the OAuth application form like this:
    Application Name: My RCloud Instance
    Homepage URL: http://127.0.0.1:8080
    Application Description: My very own RCloud instance
    Authorization callback URL: http://127.0.0.1:8080/login_successful.R

    RCloud uses GitHub for both authentication and for automatic version control of notebooks via GitHub 'gists'. The 'OAuth' applciation authentication process gives the application owner the ability to grant access to new RCloud users but this still implies that notebook code and assets reside on public GitHub servers. For improved privacy and security you may want to consider using GitHub Enterprise with your private (local) RCloud instance.

    Complete detailed instructions for installing a local RCloud instance can be found on the RCloud Installation Wiki.

  • Data Scientist Access (or Logging in)

    • As indicated in the previous section, private instances of RCloud may be setup which limit access to members of a particular community or organization. The public instance of RCloud is open to everyone. Both the public and any private installation of RCloud come with two methods for accessing notebooks: as a Guest or as a Data Scientist. Guest access is for users who are only interested in viewing and interacting with notebooks - Guests are not able to create notebooks. The purpose of Guest access is to allow Data Scientists to share their completed work as a web page (hyperlink) with users who do not have an RCloud account. Guests may view and interact with RCloud notebooks without having an RCloud account by navigating to the RCloud notebook web page using the shared hyperlink.

    Hyperlinks to individual notebooks are automatically created when a notebook is created. To create a notebook, you must first login to RCloud by clicking on the green "Log In" button on the TryIt page.


    Since RCloud stores notebooks as Github gists in order to provide automatic version control, logging into RCloud requires a Github account. If you are not already logged in to your Github account or if you don't already have one, you will be redirected to Github where you can login (Github Sign in) or set one up (Github Sign up).

    RCloud Data Scientists may obtain a hyperlink (URL) to a notebook they have created or a published notebook created by another Data Scientist by clicking on the downward arrow next to the share icon in the navigation bar. to obtain a popup menu:



    This allows Data Scientists to select the kind of URL they would like to share with Guests. The simplest form of sharing is using RCloud notebook UI "view.html"; make the selection using the popup menu and then right click on the share icon to copy the notebook hyperlink. This will create a link that will allow someone to see the notebook code and execute the notebook within the RCloud IDE.

    Note: This only works if the notebook has also been published.

    By default, users who wish to view notebooks must be logged into RCloud. If the Publish Notebook box is checked, any user who has network access to the notebook's URL will be able to view the notebook.
