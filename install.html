Creating and editing notebooks on the public RCloud instance.

The public RCloud instance is the perfect place to test drive RCloud. Anyone with a GitHub account can have full user access to this instance.

If you don't already have a GitHub account you can create one at the GitHub login page. The first time you login the the public RCloud (after setting up a GitHub account) you will be directed to GitHub to authorize access to RCloud. Click through the authorization process and on future visits to rcloud.social you will be automatically logged into your RCloud account.

Installing a local RCloud server.

The RCloud server program can easily be installed locally on any Linux machine or under OSX via MacPorts. A local instance has the advantage of granting you complete control of your notebooks and data. Local instances also affords the oppurtunity to evaluate and experiment with the RCloud infrastructure.

By default your local RCloud instance will still use GitHub for authentication. Before proceeding with an installation you will need to setup up a new 'OAuth' application in your GitHub profile. Here's how:

  1. Go to your GitHub profile page and select 'Edit Profile'.
  2. In the personal settings menu select 'OAuth applicatoins'.
  3. Select the 'Developer applications' tab.
  4. Press the 'Register a new application' button.
  5. Assuming that you are testing on your local IP address 127.0.0.1 and the default RCloud port (8080) is available fill in the OAuth application form like this:
Application Name: My RCloud Instance
Homepage URL: http://127.0.0.1:8080
Application Description: My very own RCloud instance
Authorization callback URL: http://127.0.0.1:8080/login_successful.R

RCloud uses GitHub for both authentication and for automatic version control of notebooks via GitHub 'gists'. The 'OAuth' applciation authentication process gives the application owner the ability to grant access to new RCloud users but this still implies that notebook code and assets reside on public GitHub servers. For improved privacy and security you may want to consider using GitHub Enterprise rahter with your local RCloud instance.

Complete detailed instructions for installing a local RCloud instance can be found on the RCloud Installation Wiki
