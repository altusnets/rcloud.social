There are two methods of using the public instance of RCloud; as a Guest with read only access or as an Data Scientist with developer capabilities; the following sections describe the requirements for each profile.

Guest     Read Only

Guests may view and interact with RCloud notebook analyses or dashboards without having to set up an RClound account. Click on an image below or visit our Gallery for additional examples.

Data Scientist     Read / Write / Execute

Since RCloud has built-in version control based on authorized Github gists, Data Scientist access is obtained through a GitHub account; click on the image below to access RCloud through an existing GitHub account or to be redirected to GitHub where you may set one up and authorize the RCloud app.

Log In

For help getting started, run RCloud Sample Notebooks which may be found in the notebook information panel in RCloud; to run a notebook, select the notebook name and then click the "play" triangle in the navigation bar.


Create your own notebook using the "+" key, fork notebooks using the fork icon in the navigation bar, share notebooks using the share icon (arrow in square) or begin editing by typing code in a cell; additional details are provided in the Documentation and Tutorials pages.
